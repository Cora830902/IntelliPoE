PoE Port Configuration Guide
This guide covers PoE (Power over Ethernet) port configuration for the NETGEAR M4350 series enterprise managed switches. The M4350 supports IEEE 802.3af (PoE), 802.3at (PoE+), and 802.3bt Type3/Type4 (PoE++/PoE+++) depending on the model.

Admin Mode
The Admin Mode field enables or disables PoE on a specific port. When set to Enable, the port will attempt to detect and power connected PD (Powered Device) equipment. When set to Disable, the port will not provide power regardless of other settings.
💡 Tip: Disabling unused PoE ports saves power and reduces budget consumption from the total PoE power pool.
Power Management Mode
Determines how the switch allocates PoE power budget to ports:
ModeDescriptionBudget Behavior
Dynamic Power is allocated based on actual real-time consumption measured by the PSE hardware. Budget deducted = actual consumed watts. Efficient — allows more ports to be powered.
Static Power is allocated based on the negotiated class or user-configured limit, not actual consumption. Budget deducted = configured maximum (e.g., Class 4 = 31.2W on M4350). Conservative — prevents over-allocation.
⚠️ In Static mode, M4350 v14.0.6.8 allocates power based on IEEE class values plus M4350-specific overhead: C0=16.2W, C1=4.2W, C2=7.4W, C3=16.2W, C4=31.2W, C5=48.6W, C6=65W, C7=81.8W, C8=97W.
Detection Type
Controls which IEEE detection method the PSE uses to identify valid PD devices. Invalid or unsupported devices will not be powered.
OptionDescriptionSifos Support
4ptdot3afStandard IEEE 802.3af 4-point detection. Most compatible. Recommended for all standard PoE devices.✅ Supported
4ptdot3af + legacyExtended detection — tries standard first, falls back to pre-standard detection for older proprietary devices.✅ Supported
legacyPre-802.3af proprietary detection only. Used for very old pre-standard PoE devices (rare). Not recommended.❌ Not Supported by Sifos
⚠️ Selecting "legacy" detection on Sifos PowerSync Interactive 5 is not supported. The system will automatically revert to "4ptdot3af" if selected.
Power Limit Type
Controls the maximum power the switch will deliver to a port, providing a ceiling above which power will be cut:
TypeDescription
NoneNo software-imposed power limit. The hardware physical limit applies.
ClassPower limit set according to IEEE 802.3 class negotiation result. PSE will not deliver more than the class maximum.
UserAdministrator-defined wattage limit (1–99W). Overrides class-based limits. Useful for budget control.
Threshold Power
The Threshold Power setting defines a soft warning threshold in watts. When a port's consumed power exceeds this value, a trap/alert is generated.
⚠️ Known Issue M4350-96: On M4350-96 variants, the Threshold Power is treated as a soft limit — the switch will continue supplying power beyond the threshold value as long as the Total PoE Power Available budget is not exceeded. This is expected behavior and should be documented as a Known Issue in test reports.
💡 Tip: Set Threshold Power slightly below the maximum expected device consumption for early warning without service interruption.

Port Priority
When the total PoE power demand exceeds the available budget, the switch uses port priority to determine which ports remain powered:
PriorityDescription
CriticalHighest priority. Port retains power even when budget is tight. Use for life-safety devices (cameras, access control).
HighMedium-high priority. Powered before Low ports.
LowDefault priority. First ports to be powered down under budget stress.
